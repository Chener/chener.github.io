jQuery Mobile 开发

jQuery Mobile（以下简称 jqm）其实挺好用。

对于易百里现阶段团队而言，手机端开发 Phonegap 结合 jqm(v1.4.5) 是最好的方式。Ionic Framework 表现会更好，但要求 Angular.js 基础。有关 Phonegap 的使用，详见Phonegap 开发。

jqm 项目文件结构

  • 目前对多角色系统的权限分配比较原始，采用角色与文件夹对应的方式。
  • 开发时会有很多”碎片化”的 css``js 文件，PC 端需要在上线时打包提高访问速度，Phone 端貌似不需要（待）。
  • 欢迎提出修改意见。

css/
..lib/ // 公共库或框架提供的 css
….images/ // 公共库或框架提供 css 包含的图片
..common.css // 很多页面都会引用的公用样式
..xxx // 各个分页(包括 index.html)需要的独有 css 以及一些少数页面通用的 UI 模块 css
js/
..lib/ // 同 css，只是没有 images/
..common.js
img/ // 系统生成或者需要展示的图片（其它多媒体文件以此为例）
..xxx/ // 可按系统模块划分图片文件夹
xxx
A/ // 角色 A 所在文件夹
..a/ // 角色 A 下所包含的功能模块 a, a 下将有至少一个 HTML 页面，其只有一个 data-role = "page" 部分，页面对应的 js 将在页面最后引用。（移除 DOM 既能移除对应 js 运行结果，防止内存溢出，待）
..b/ // …
..xxx
..A.html // 角色 A 的主页，因为采取 jqm 默认页面转换与加载方式，所以除了库文件还将引用角色下所有页面的独有 css 以及 common.css。
index.html // 登录页面，依据登录用户的角色跳转(或者加载然后移除登录页面 DOM)至对应角色文件夹下的角色主页。
